Database Schema

Tables

The database schema include the following tables: employees, jobs, managers, departments, job_histories, attendance_logs, savings, payrolls, satisfaction_scores, performance_scores, and attritions.

Employees

Create table to represent employees in the company.

CREATE TABLE "employees" (
    "id" INTEGER,
    "first_name" TEXT NOT NULL,
    "last_name" TEXT NOT NULL,
    "SSS" TEXT NOT NULL UNIQUE CHECK(length("SSS") = 10),
    "birth_date" NUMERIC NOT NULL,
    "sex_at_birth" TEXT NOT NULL CHECK("sex_at_birth" IN ('female', 'male')),
    "email" TEXT NOT NULL UNIQUE,
    "contact_number" TEXT NOT NULL UNIQUE CHECK(length("contact_number") = 11),
    "civil_status" TEXT NOT NULL CHECK("civil_status" IN ('single', 'married', 'widowed', 'legally separated')),
    "start_date" NUMERIC NOT NULL,
    "end_date" NUMERIC,
    "job_id" INTEGER,
    "manager_id" INTEGER,
    "department_name" TEXT,
    PRIMARY KEY("id"),
    FOREIGN KEY("job_id") REFERENCES "jobs"("id"),
    FOREIGN KEY("manager_id") REFERENCES "managers"("id"),
    FOREIGN KEY("department_name") REFERENCES "departments"("name") ON UPDATE CASCADE
);

CREATE TABLE "jobs" (

Triggers1

The database schema include the following triggers: tardiness, shift_end, new_job, savings_adjustment, payroll_calculation, performance_average, satisfaction_average, and soft_delete_employees.

Tardiness

Upon time in, this trigger will update the tardy column depending on the time_in value. The employee is considered late when they are more than 30 minutes late (on-time is 08:00:00). The "type" is NULL in the WHERE clause prevents the trigger from updating on INSERTs with absents and paid leaves.

CREATE TRIGGER "tardiness"
AFTER INSERT ON "attendance_logs"
FOR EACH ROW
BEGIN
    UPDATE "attendance_logs"
    SET "tardy" =
        CASE
            WHEN (
                SELECT ROUND (
                    (julianday(NEW."date"||' 08:00:00') - julianday(NEW."time_in")) * 24
                )
                FROM "attendance_logs"
                WHERE "employee_id" = NEW."employee_id" AND "date" = NEW."date"
            ) < 0
                THEN 1
            ELSE 0
        END
    WHERE "employee_id" = NEW."employee_id" AND "date" = NEW."date" AND "type" IS NULL;
END;

Shift End

Upon time out, the hours worked will be calculated. The calculated hours_worked will be used to determine if the shift is undertime, normal, or overtime. This will then calculate the overtime_hours. The normal shift is 8 hours, give or take 30 minutes.

CREATE TRIGGER "shift_end"
AFTER UPDATE OF "time_out" ON "attendance_logs"
FOR EACH ROW
BEGIN
    UPDATE "attendance_logs"
    SET "hours_worked" = ROUND((julianday(NEW."time_out") - julianday(OLD."time_in"))*24)
    WHERE "employee_id" = OLD."employee_id" AND "date" = OLD."date";

    UPDATE "attendance_logs"
    SET "type" =
        CASE
            WHEN "hours_worked" > 8 THEN 'overtime'
            WHEN "hours_worked" = 8 THEN 'normal'
            ELSE 'undertime'
        END
    WHERE "employee_id" = OLD."employee_id" AND "date" = OLD."date";

    UPDATE "attendance_logs"
    SET "overtime_hours" = "hours_worked" - 8
    WHERE "employee_id" = OLD."employee_id" AND "date" = OLD."date" AND "type" = 'overtime';
END;

New Job

This is triggered when an employee is assigned to a new job (new hire or old employee). If applicable, the pay changes is also recorded. The trigger also enrolls the employee to a savings plan and updates with every subsequent job changes. This works along with the savings_adjustment trigger.

CREATE TRIGGER "new_job"
AFTER INSERT ON "job_histories"
FOR EACH ROW
BEGIN
    UPDATE "employees"
    SET "job_id" = (
        SELECT "job_id"
        FROM "job_histories"
        WHERE "employee_id" = NEW."employee_id"
        ORDER BY "start_date" DESC LIMIT 1
    )
    WHERE "id" = NEW."employee_id";

    INSERT INTO "savings"
    ("date", "employee_id", "salary_change", "previous_rate")
    VALUES
    (NEW."start_date", NEW."employee_id", NEW."raise/cut",
        (SELECT "adjusted_rate"
        FROM "savings"
        WHERE "employee_id" = NEW."employee_id"
        ORDER BY "date" DESC
        LIMIT 1
        )
    );
END;

Savings Adjustment

Based on the changes made by the new_job trigger, savings rate will be adjusted. The adjustment is based on the 10% of the salary_change. The starting savings rate is 3% and the maximum is 18%.

CREATE TRIGGER "savings_adjustment"
AFTER INSERT ON "savings"
FOR EACH ROW
BEGIN
    UPDATE "savings"
    SET "adjusted_rate" =
        CASE
            WHEN "previous_rate" IS NULL THEN 0.03
            WHEN (
                SELECT COALESCE("previous_rate", 0) + (COALESCE("salary_change", 0) * 0.1)
                FROM "savings"
                WHERE "employee_id" = NEW."employee_id"
                ORDER BY "date" DESC
                LIMIT 1
            ) >= 0.18
                THEN 0.18
            ELSE (
                SELECT COALESCE("previous_rate", 0) + (COALESCE("salary_change", 0) * 0.1)
                FROM "savings"
                WHERE "employee_id" = NEW."employee_id"
                ORDER BY "date" DESC
                LIMIT 1
            )
        END
    WHERE "employee_id" = NEW."employee_id" AND "date" = NEW."date";
END;

Payroll Calculation

This trigger is initialized by an INSERT, which collects the necessary information to calculate the payroll. This calculates the gross_pay, SSS_deduction (savings plan), and the taxable_pay. The pay for overtime hours is 125% of the hourly rate. The pay for paid leave is for the full 8 hours. The SSS deduction is not used for tax calculation.

CREATE TRIGGER "payroll_calculation"
AFTER INSERT ON "payrolls"
FOR EACH ROW
BEGIN
    UPDATE "payrolls"
    SET
    "gross_pay" = (
        SELECT ("hourly_rate" * "regular_hours") + ("hourly_rate" * 1.25 * "overtime_hours") +
            ("paid_leave" * 8 * "hourly_rate")
        FROM "payrolls"
        WHERE "employee_id" = NEW."employee_id" AND "date" = NEW."date"
    ),
    "SSS_deduction" = (
        SELECT (("hourly_rate" * "regular_hours") + ("hourly_rate" * 1.25 * "overtime_hours") +
            ("paid_leave" * 8 * "hourly_rate")) * (
                SELECT "adjusted_rate"
                FROM "savings"
                WHERE "employee_id" = NEW."employee_id"
                ORDER BY "date" DESC
                LIMIT 1
                )
        FROM "payrolls"
        WHERE "employee_id" = NEW."employee_id" AND "date" = NEW."date"
    ),
    "taxable_pay" = (
        SELECT (("hourly_rate" * "regular_hours") + ("hourly_rate" * 1.25 * "overtime_hours") +
            ("paid_leave" * 8 * "hourly_rate")) * (1 - (
                SELECT "adjusted_rate"
                FROM "savings"
                WHERE "employee_id" = NEW."employee_id"
                ORDER BY "date" DESC
                LIMIT 1
                ))
        FROM "payrolls"
        WHERE "employee_id" = NEW."employee_id" AND "date" = NEW."date"
    )
    WHERE "employee_id" = NEW."employee_id" AND "date" = NEW."date";
END;

Performance Average

Based on the performance evaluation in different measures, the average performance score is calculated.

CREATE TRIGGER "performance_average"
AFTER INSERT ON "performance_scores"
FOR EACH ROW
BEGIN
    UPDATE "performance_scores"
    SET "average_score" = (
        SELECT ROUND(
            (
                ("productivity_score" + "quality_score" + "teamwork_score" + "initiative_score" + "leadership_score")
            /5)
        , 2)
        FROM "performance_scores"
        WHERE "employee_id" = NEW."employee_id" AND "date" = NEW."date"
    )
    WHERE "employee_id" = NEW."employee_id" AND "date" = NEW."date";
END;

Satisfaction Average

Based on the satisfaction ratings in different areas, the average satisfaction score is calculated.

CREATE TRIGGER "satisfaction_average"
AFTER INSERT ON "satisfaction_scores"
FOR EACH ROW
BEGIN
    UPDATE "satisfaction_scores"
    SET "average_score" = (

Soft Delete Employees

This trigger updates the end_date of an employee in the employees table. The end_date serves as a marking of soft deletion.

CREATE TRIGGER "soft_delete_employees"
AFTER INSERT ON "attritions"
FOR EACH ROW
BEGIN
    UPDATE "employees"
    SET "end_date" = NEW."end_date"
    WHERE "id" = NEW."employee_id";
END;

Views

The database schema include the following views: monthly_attendance, manager_1, employee_evaluation,and attrition_analysis.

Monthly Attendance

This view shows an overview of the monthly attendance records. This shows counts of shift by type and counts of tardiness.

CREATE VIEW "monthly_attendance" AS
WITH "undertimes" AS (
    SELECT
        "employee_id",
        COUNT(*) AS "undertime_count"
    FROM "attendance_logs"
    WHERE strftime('%m', "date") LIKE (strftime('%m', DATE('now')))||'%'
        AND "type" = 'undertime'
    GROUP BY "employee_id"
),
"normals" AS (
    SELECT
        "employee_id",
        COUNT(*) AS "normal_count"
    FROM "attendance_logs"
    WHERE strftime('%m', "date") LIKE (strftime('%m', DATE('now')))||'%'
        AND "type" = 'normal'
    GROUP BY "employee_id"
),
"overtimes" AS (
    SELECT
        "employee_id",
        COUNT(*) AS "overtime_count"
    FROM "attendance_logs"
    WHERE strftime('%m', "date") LIKE (strftime('%m', DATE('now')))||'%'
        AND "type" = 'overtime'
    GROUP BY "employee_id"
),
"absents" AS (
    SELECT
        "employee_id",
        COUNT(*) AS "absent_count"
    FROM "attendance_logs"
    WHERE strftime('%m', "date") LIKE (strftime('%m', DATE('now')))||'%'
        AND "type" = 'absent'
    GROUP BY "employee_id"
),
"paid_leaves" AS (
    SELECT
        "employee_id",
        COUNT(*) AS "paid_leave_count"
    FROM "attendance_logs"
    WHERE strftime('%m', "date") LIKE (strftime('%m', DATE('now')))||'%'
        AND "type" = 'paid leave'
    GROUP BY "employee_id"
    ),
"tardies" AS (
    SELECT
        "employee_id",
        SUM("tardy") AS "tardy_count"
    FROM "attendance_logs"
    WHERE strftime('%m', "date") LIKE (strftime('%m', DATE('now')))||'%'
    GROUP BY "employee_id"
)
SELECT
    "employees"."id" AS "employee_id",
    "employees"."job_id",
    "employees"."manager_id",
    "employees"."department_name",
    IFNULL("undertimes"."undertime_count", 0) AS "undertime_count",
    IFNULL("normals"."normal_count", 0) AS "normal_count",
    IFNULL("overtimes"."overtime_count", 0) AS "overtime_count",
    IFNULL("absents"."absent_count", 0) AS "absent_count",
    IFNULL("paid_leaves"."paid_leave_count", 0) AS "paid_leave_count",
    "tardies"."tardy_count"
FROM "employees"
LEFT JOIN "undertimes" ON
"employees"."id" = "undertimes"."employee_id"
LEFT JOIN "normals" ON
"employees"."id" = "normals"."employee_id"
LEFT JOIN "overtimes" ON
"employees"."id" = "overtimes"."employee_id"
LEFT JOIN "absents" ON
"employees"."id" = "absents"."employee_id"
LEFT JOIN "paid_leaves" ON
"employees"."id" = "paid_leaves"."employee_id"
LEFT JOIN "tardies" ON
"employees"."id" = "tardies"."employee_id";

Manager 1

This is a sample view for the managers. This shows the satisfaction of employees under a particular manager. The employee_id is hidden from this view to encourage honest feedbacks. This report is for the current year average only.

CREATE VIEW "manager_1" AS
WITH "satisfactions" AS (
    SELECT
        "employee_id",
        AVG("compensation_score") AS "compensation_satisfaction",
        AVG("workload_score") AS "workload_satisfaction",
        AVG("opportunity_score") AS "opportunity_satisfaction",
        AVG("environment_score") AS "environment_satisfaction",
        AVG("leadership_score") AS "leadership_satisfaction",
        AVG("teamwork_score") AS "teamwork_satisfaction",
        AVG("average_score") AS "average_satisfaction"
    FROM "satisfaction_scores"
    WHERE strftime('%Y', "date") LIKE (strftime('%Y', DATE('now')))||'%'
    GROUP BY "employee_id"
)
SELECT
    "satisfactions"."compensation_satisfaction",
    "satisfactions"."workload_satisfaction",
    "satisfactions"."opportunity_satisfaction",
    "satisfactions"."environment_satisfaction",
    "satisfactions"."leadership_satisfaction",
    "satisfactions"."teamwork_satisfaction",
    "satisfactions"."average_satisfaction"
FROM "employees"
LEFT JOIN "satisfactions" ON
"employees"."id" = "satisfactions"."employee_id"
WHERE "employees"."manager_id" = 1
    AND "employees"."id" <> (SELECT "employee_id" FROM "managers" WHERE "id" = 1);

Employee Evaluation

This view shows the average performance evaluation of employees. This report is for the current year average only.

CREATE VIEW "employee_evaluation" AS
SELECT
    "employee_id",
    AVG("productivity_score") AS "productivity_performance",
    AVG("quality_score") AS "quality_performance",
    AVG("teamwork_score") AS "teamwork_performance",
    AVG("initiative_score") AS "initiative_performance",
    AVG("leadership_score") AS "leadership_performance",
    AVG("average_score") AS "average_performance"
FROM "performance_scores"
WHERE strftime('%Y', "date") LIKE (strftime('%Y', DATE('now')))||'%'
GROUP BY "employee_id";

Attrition Analysis

This view is useful for attrition analysis. This view makes it easy to later export the data in a CSV file for statistical analysis. This shows only the date for the current year.

CREATE VIEW "attrition_analysis" AS
WITH "performances" AS (
    SELECT
        "employee_id",
        AVG("productivity_score") AS "productivity_performance",
        AVG("quality_score") AS "quality_performance",
        AVG("teamwork_score") AS "teamwork_performance",
        AVG("initiative_score") AS "initiative_performance",
        AVG("leadership_score") AS "leadership_performance",
        AVG("average_score") AS "average_performance"
    FROM "performance_scores"
    WHERE strftime('%Y', "date") LIKE (strftime('%Y', DATE('now')))||'%'
    GROUP BY "employee_id"
),
"satisfactions" AS (
    SELECT
        "employee_id",
        AVG("compensation_score") AS "compensation_satisfaction",
        AVG("workload_score") AS "workload_satisfaction",
        AVG("opportunity_score") AS "opportunity_satisfaction",
        AVG("environment_score") AS "environment_satisfaction",
        AVG("leadership_score") AS "leadership_satisfaction",
        AVG("teamwork_score") AS "teamwork_satisfaction",
        AVG("average_score") AS "average_satisfaction"
    FROM "satisfaction_scores"
    WHERE strftime('%Y', "date") LIKE (strftime('%Y', DATE('now')))||'%'
    GROUP BY "employee_id"
)
SELECT
    "employees"."id",
    IIF("attritions"."employee_id" IS NOT NULL, 1, 0) AS "attrition",
    CAST(
        ROUND(
                (
                    (julianday('now') - julianday("employees"."start_date"))
                    /(365/12)
                )
        , 0)
    AS INT) AS "tenure_in_months",
    DATE('now') - "employees"."birth_date" AS "age",
    "employees"."sex_at_birth",
    "employees"."civil_status",
    "performances"."productivity_performance",
    "performances"."quality_performance",
    "performances"."teamwork_performance",
    "performances"."initiative_performance",
    "performances"."leadership_performance",
    "performances"."average_performance",
    "satisfactions"."compensation_satisfaction",
    "satisfactions"."workload_satisfaction",
    "satisfactions"."opportunity_satisfaction",
    "satisfactions"."environment_satisfaction",
    "satisfactions"."leadership_satisfaction",
    "satisfactions"."teamwork_satisfaction",
    "satisfactions"."average_satisfaction",
    "employees"."job_id",
    "employees"."manager_id",
    "employees"."department_name"
FROM "employees"
LEFT JOIN "attritions" ON
"employees"."id" = "attritions"."employee_id"
LEFT JOIN "performances" ON
"employees"."id" = "performances"."employee_id"
LEFT JOIN "satisfactions" ON
"employees"."id" = "satisfactions"."employee_id";

Indexes

Indexes are not strictly necessary for this database. As this is for internal use only and only occasionally uses SELECT queries, the space saving trumps the time saving. There is also trade-off in speed with relatively more frequent operations involving UPDATE and INSERT. Thus, this database schema only includes one index: names.

Names

CREATE INDEX "names"
ON "employees" ("first_name", "last_name");
